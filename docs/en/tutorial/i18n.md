## How to Modify Fixed Text/Language Files {#theme-fixed-text-language-files}

Go to the Halo CMS deployment directory and find the `themes`/`howiehz-higan`/`i18n` folder. Inside, find the `properties` file for the corresponding language, modify and save it (for example, Simplified Chinese is `zh_Hans.properties`).

## Sync Cookie Language Based on Page Content Language

This theme enables [Sync Cookie Language Based on Page Content Language](/en/guide/theme-configuration#sync-cookie-language-based-on-page-content-language) by default.

When enabled, if a page is visited without a `?language=` parameter and the browser's `language` Cookie differs from the current page language[^current-page-language], the theme synchronizes the `language` Cookie to the current page language[^current-page-language] and refreshes the page after the Cookie is set successfully.

If the visit triggers browser language-based auto redirect, the theme redirects to the target page with `?language=...` first and does not perform Cookie synchronization on the current page.

This prevents [theme fixed text](#theme-fixed-text-language-files) from continuing to use the previous Cookie language and becoming inconsistent with the current page language[^current-page-language]. If you do not want Cookie synchronization to cause an extra refresh, you can disable this option.

## Browser Language-Based Auto Redirect Guide

This theme has comprehensive support for language-based auto redirect. You can enable [Browser Language-Based Auto Redirect](/en/guide/theme-configuration#auto-redirect-based-on-browser-language).

::: warning

When the URL already contains a `?language=` parameter, or when the current [post](/en/guide/metadata-configuration#post-page-language), [category](/en/guide/metadata-configuration#category-page-language), [tag](/en/guide/metadata-configuration#tag-page-language), or [page](/en/guide/metadata-configuration#page-language) has a "Page Language" metadata marker, browser language-based auto redirect will not run.

:::

::: tip

Press `F12` (or `Fn+F12`) to open the developer console, select the console tab.
Enter `navigator.language` and press Enter to view your browser's `navigator.language` value.

:::

After enabling this option, [Allowed Target Locale Language Code List](/en/guide/theme-configuration#allowed-target-language-code-list-for-redirects) should be set in the following format.  
Note: When [Multilingual Function Prefix Match Mode](/en/guide/theme-configuration#multilingual-function-prefix-matching-mode) is disabled, matching follows the browser language candidate path order. When it is enabled, matching follows the list below from top to bottom.

| Allowed Target Locale Language Code List |
| ---------------------------------------- |
| `zh-CN`                                  |
| `en-US`                                  |

If the matched target language is equivalent to the [Default Page Language](/en/guide/theme-configuration#default-page-language), the theme treats the current page without `?language=` as already being the target language and does not redirect to a URL with `?language=`.

"Equivalent" here means the languages can be treated as the same target under the browser language candidate rules. For example, `zh-CN` can be treated as equivalent to `zh-Hans`, but `zh-Hans` and `zh-Hant` are not treated as equivalent just because both can fall back to `zh`.

::: tip

For easier configuration, auto redirect converts the browser language into candidate language codes before matching the list. Chinese language candidates use the following paths:

| `navigator.language` value  | Candidate path                                                    |
| --------------------------- | ----------------------------------------------------------------- |
| `zh-CN` / `zh-SG` / `zh-MY` | `zh-Hans-{REGION}` -> `zh-Hans` -> original language code -> `zh` |
| `zh-Hans`                   | `zh-Hans` -> `zh`                                                 |
| `zh-TW` / `zh-HK` / `zh-MO` | `zh-Hant-{REGION}` -> `zh-Hant` -> original language code -> `zh` |
| `zh-Hant`                   | `zh-Hant` -> `zh`                                                 |
| `zh`                        | `zh-Hans` -> `zh`                                                 |

**Summary**: In most cases, configuring `zh-Hans` and `zh-Hant` is enough to cover Chinese regional language tags.

:::

After enabling [Multilingual Function Prefix Match Mode](/en/guide/theme-configuration#multilingual-function-prefix-matching-mode), language codes in the [Allowed Target Locale Language Code List](/en/guide/theme-configuration#allowed-target-language-code-list-for-redirects) can match browser language candidates by prefix instead of requiring an exact match.  
Example: `zh` below can match Chinese candidates such as `zh-Hans` and `zh-Hant`.

| Allowed Target Locale Language Code List |
| ---------------------------------------- |
| `zh`                                     |
| `en`                                     |

## Multilingual Menu Usage Guide

This theme has comprehensive multilingual menu support. You can enable [Multilingual Menu Support](/en/guide/theme-configuration#multilingual-menu-support).

The multilingual menu matches top-level menu names against the current page language[^current-page-language].

[^current-page-language]: The current page language refers to the `lang` attribute on the page's `<html>` element, and its setting order follows [Page Language Setting Priority](/en/reference/faq#page-language-setting-priority).

After enabling this option, the main menu should be set in the following format. (Note: The `zh-CN` item can be a custom link with URL `/` and name `zh-CN`. The name is the key setting; other settings don't affect matching)

| Top-level menu item | Second-level menu items |
| ------------------- | ----------------------- |
| `zh-CN`             | 首页<br>关于            |
| `en-US`             | Home<br>About           |

After enabling [Multilingual Function Prefix Match Mode](/en/guide/theme-configuration#multilingual-function-prefix-matching-mode), top-level menu names can match the current page language[^current-page-language] by prefix instead of requiring an exact match.  
Example: when the current page language[^current-page-language] is `zh-Hans` or `zh-Hant`, the `zh` menu below can match both.

| Top-level menu item | Second-level menu items |
| ------------------- | ----------------------- |
| `zh`                | 首页<br>关于            |
| `en`                | Home<br>About           |

The menu with a name that **exactly matches** the [Default Page Language](/en/guide/theme-configuration#default-page-language) setting value will be used as the default menu, which will be displayed when no successful match is found.
