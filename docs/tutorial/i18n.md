## 如何正确设定 HTML 页面语言

设定优先级：请查阅[页面语言设定优先级](/reference/faq#页面语言设定优先级)。

1. 正确设定主题设置中的[默认页面语言](/guide/theme-configuration#默认页面语言)
2. [文章](/guide/metadata-configuration#文章页面语言)、[分类](/guide/metadata-configuration#分类页面语言)、[标签](/guide/metadata-configuration#标签页面语言)、[页面](/guide/metadata-configuration#页面语言)与[默认页面语言](/guide/theme-configuration#默认页面语言)不同的，前往对应设置正确设定其元数据。

## 如何修改页面固定文字/修改语言文件 {#theme-fixed-text-language-files}

请到 Halo CMS 部署目录下，找到 `themes`/`howiehz-higan`/`i18n` 文件夹。在里面找到对应语言的 `properties` 文件，修改保存即可（如简体中文是 `zh_Hans.properties`）。

## 根据页面内容语言同步 Cookie 语言

本主题默认开启[是否根据页面内容语言同步 Cookie 语言](/guide/theme-configuration#是否根据页面内容语言同步-cookie-语言)。

开启后，若无 `?language=` 参数访问页面，且浏览器中的 `language` Cookie 与当前页面语言[^current-page-language]不一致，主题会将 `language` Cookie 同步为当前页面语言[^current-page-language]，并在设置成功后刷新一次页面。

若本次访问会触发浏览器按语言自动跳转，主题会先跳转到带有 `?language=...` 的目标页面，不会在当前页面执行 Cookie 同步。

这可避免[主题固定文字](#theme-fixed-text-language-files)继续按上一次 Cookie 语言显示，导致当前页面语言[^current-page-language]与主题文字不一致。若不希望同步 Cookie 造成额外刷新，可关闭此项。

## 浏览器按语言自动跳转使用指南

本主题有完善的按语言自动跳转支持，可启用[浏览器按语言自动跳转](/guide/theme-configuration#浏览器按语言自动跳转)。

::: warning

若 URL 已包含 `?language=` 参数，或当前[文章](/guide/metadata-configuration#文章页面语言)、[分类](/guide/metadata-configuration#分类页面语言)、[标签](/guide/metadata-configuration#标签页面语言)、[页面](/guide/metadata-configuration#页面语言)已设置“页面语言”元数据标记，则不会执行浏览器按语言自动跳转。

:::

::: tip

按 `F12`（或 `Fn+F12`）打开开发者控制台，上面选择控制台（`console`）。
输入 `navigator.language` 后回车即可查看你浏览器的 `navigator.language` 值。

:::

启用此选项后，[允许跳转的目标区域语言代码列表](/guide/theme-configuration#允许跳转的目标区域语言代码列表)应设置形如以下形式。  
注：匹配顺序从上到下。

| 允许跳转的目标区域语言代码列表 |
| ------------------------------ |
| `zh-CN`                        |
| `en-US`                        |

::: tip

为方便配置，自动跳转会先将浏览器语言转换为候选语言代码后再匹配列表。中文语言的候选路径如下：

| `navigator.language` 值     | 候选路径                                                |
| --------------------------- | ------------------------------------------------------- |
| `zh-CN` / `zh-SG` / `zh-MY` | `zh-Hans-{REGION}` -> `zh-Hans` -> 原始语言代码 -> `zh` |
| `zh-Hans`                   | `zh-Hans` -> `zh`                                       |
| `zh-TW` / `zh-HK` / `zh-MO` | `zh-Hant-{REGION}` -> `zh-Hant` -> 原始语言代码 -> `zh` |
| `zh-Hant`                   | `zh-Hant` -> `zh`                                       |
| `zh`                        | `zh-Hans` -> `zh`                                       |

**总结**：一般只需要配置 `zh-Hans` 和 `zh-Hant` 即可适配中文区域语言标签。

:::

启用[多语言功能前缀匹配模式](/guide/theme-configuration#多语言功能前缀匹配模式)后，[允许跳转的目标区域语言代码列表](/guide/theme-configuration#允许跳转的目标区域语言代码列表)中的语言代码可按浏览器语言候选前缀匹配，而无需完全匹配。  
例：下面的 `zh` 可匹配 `zh-Hans`、`zh-Hant` 等中文候选。

| 允许跳转的目标区域语言代码列表 |
| ------------------------------ |
| `zh`                           |
| `en`                           |

## 多语言菜单使用指南

本主题有完善的多语言菜单支持，可启用[多语言菜单支持](/guide/theme-configuration#多语言菜单支持)。

多语言菜单根据当前页面语言[^current-page-language]匹配一级菜单名称。

[^current-page-language]: 当前页面语言指代页面 `<html>` 的 `lang` 属性，设定顺序遵循[页面语言设定优先级](/reference/faq#页面语言设定优先级)。

启用此选项后，主菜单应设置形如以下形式。（注：`zh-CN` 项可选择自定义链接，链接为 `/`，名称为 `zh-CN`。其中名称为关键设置，其他不影响匹配）

| 一级菜单项 | 二级菜单项    |
| ---------- | ------------- |
| `zh-CN`    | 首页<br>关于  |
| `en-US`    | Home<br>About |

启用[多语言功能前缀匹配模式](/guide/theme-configuration#多语言功能前缀匹配模式)后，将允许一级菜单名称匹配当前页面语言[^current-page-language]的前缀，而无需完全匹配。  
例：当前页面语言[^current-page-language]为 `zh-Hans` 或 `zh-Hant` 时，下面的 `zh` 菜单均可匹配。

| 一级菜单项 | 二级菜单项    |
| ---------- | ------------- |
| `zh`       | 首页<br>关于  |
| `en`       | Home<br>About |

与[默认页面语言](/guide/theme-configuration#默认页面语言)设置值**完全同名**的将被作为默认菜单，默认菜单将在没有成功匹配的时候显示。
