# 配置总览

所有主题设置均位于 Halo 后台的「外观 → 主题 → 设置」。本章节按配置分组逐项说明默认值、作用及注意事项，并给出与其他功能的联动提示。

> **说明**：文中出现的设置路径以 `分组 → 选项名称` 表示，例如 `全局 → 默认页面语言`。

## 使用建议

- 保存设置后刷新前台才能看到最新效果；涉及样式的开关建议清空浏览器缓存后再观察。
- 开启/关闭某些开关后需要再次打开主题设置页面才能看到新增字段（Halo 的表单渲染机制所致）。
- 重复项（Repeater）支持拖拽排序，表单中的“值”字段均支持国际化文本（若启用多语言）。

## 全局

### 默认页面语言
- **路径**：`全局 → 默认页面语言`
- **默认值**：`zh`
- **说明**：决定站点未显式设置语言时的 HTML `lang` 属性。可配合 URL 上的 `?lang=` 参数强制切换语言。

### 多语言功能前缀匹配模式
- **路径**：`全局 → 多语言功能前缀匹配模式`
- **默认**：开启
- **说明**：启用后，浏览器的 `navigator.language` 只要与配置项前缀相同即可匹配。例如 `zh` 可以匹配 `zh_CN`、`zh_TW`。

### 浏览器语言自动跳转
- **路径**：`全局 → 浏览器语言自动跳转`
- **默认**：关闭
- **说明**：开启后，根据客户端语言自动切换页面语言。配合下方的“允许跳转的目标区域语言代码列表”限定可跳转的目标。

### 允许跳转的目标区域语言代码列表
- **路径**：`全局 → 允许跳转的目标区域语言代码列表`
- **说明**：在启用自动跳转后可配置优先级顺序。每项填写完整的语言代码，例如 `zh_CN`、`en_US`。

### 多语言菜单支持
- **路径**：`全局 → 多语言菜单支持`
- **默认**：关闭
- **说明**：开启后可为不同语言维护不同的导航菜单。详细操作见《[多语言菜单指南](./content-components.md#菜单与多语言)》。

### CSP: upgrade-insecure-requests
- **路径**：`全局 → CSP:upgrade-insecure-requests`
- **默认**：关闭
- **作用**：对 HTTPS 页面自动升级所有 HTTP 子资源请求，避免混合内容。开启前需确认外部资源均支持 HTTPS。

### 仅允许使用指定域名访问
- **路径**：`全局 → 仅允许使用指定域名访问`
- **默认**：关闭
- **说明**：启用后可防止镜像站点盗用流量。需配合白名单、目标链接及“是否保留路径和查询参数”一起使用。所有域名与链接需使用 **Base64 编码后的值**。

### 自定义资源位置地址
- **路径**：`全局 → 自定义资源位置地址`
- **默认**：关闭
- **说明**：将主题静态资源切换到自定义 CDN。开启后可分别设置 `instant.page` 资源地址。

### instant.page 支持
- **路径**：`全局 → instant.page 支持`
- **默认**：开启
- **说明**：启用预读取，大幅提升首触延迟。若站点不需要预加载或者希望降低服务器压力可关闭。

### Mermaid 支持
- **路径**：`全局 → Mermaid 支持`
- **默认**：关闭
- **说明**：开启后将自动在文章页注入 Mermaid 渲染脚本。可自定义选择器、脚本地址与 Mermaid 配置，详见《[进阶功能](./advanced.md#mermaid)》。

## 总体样式

### 自定义字体文件
- **路径**：`总体样式 → 启用自定义字体文件`
- **默认**：关闭
- **说明**：上传字体后可指定字体名称。若名称留空，浏览器将始终下载网络字体而不读取本地缓存。

### 配色方案
- **路径**：`总体样式 → 配色方案`
- **默认值**：`跟随系统 - 绿`
- **说明**：决定初次加载时的配色。若选择“自定义配色”，需先在“自定义配色方案”中定义识别码。

### 自定义配色方案
- **路径**：`总体样式 → 自定义配色方案`
- **说明**：可添加多个配色并通过识别码引用。支持原始 CSS 输出模式或仅填写 CSS 变量。示例参见主仓库 README 的配色段落。

### 浅色/深色模式切换按钮
- **路径**：`总体样式 → 浅色/深色模式切换按钮`
- **默认**：关闭
- **说明**：开启后会在页眉添加模式切换器，并展示自动、浅色、深色的切换顺序。可分别指定各模式使用的配色方案，支持记忆用户选择。

### 字体大小
- **路径**：`总体样式 → 字体大小`
- **默认**：`small`
- **说明**：调整全站基础字号。推荐在移动端做兼容性测试。

### 内容区域宽度设置
- **相关选项**：
  - `总体样式 → 自定义内容区域最大宽度`（默认开启，值 `48rem`）
  - `总体样式 → 自定义内容区域最小宽度`
  - `总体样式 → 强制应用内容区域最小宽度`
  - `总体样式 → 自定义内容区域宽度属性`
- **说明**：这些选项控制布局张力。若关闭最大宽度设置，布局将采用上游主题的自适应逻辑，文本较少时可能偏左，可配合最小宽度与 `fit-content` 调整。

### 页眉头像
- **路径**：`总体样式 → 是否显示页眉头像`
- **默认**：开启
- **说明**：可上传自定义头像，并选择圆形/灰度效果。

### 菜单与页脚相关
- `菜单中随机文章项`：在主菜单追加“随机文章”入口。
- `是否显示页眉菜单`：隐藏头部导航条。
- `是否显示页码`：控制文章列表分页控件。
- `页面底部站点统计信息`：可配置总计、图标及多语言包裹格式。支持阅读量、文章数、点赞数、评论数、分类数、总字数（需 API 扩展插件）。
- `页面底部主题信息` 与 `页面底部版权信息`：可自由组合显示内容、署名与 Halo 品牌。
- `强制页脚、页码在页面底部`：让页面高度不足时也固定页脚。
- `页面底部菜单` 与 `页面最底部内容`：自定义额外的 Footer 内容，其中“多语言页面最底部内容”支持按语言配置 HTML 片段。

### 排版增强
- `为三级标题添加下划线`
- `引用块保留空行`
- `引用块前/后添加引号`
- `表格每行底部的表格线` + 宽度设置
- `标题/段落 上下边距倍率`

这些选项主要影响 Markdown 渲染效果。若你在 `content-components.md` 中启用了主题专属组件，建议保持引用块及表格相关选项为默认。

## 首页样式

### 主页 HTML 标题
- **路径**：`首页样式 → 主页 HTML 标题`
- **说明**：覆盖 Halo 基本设置中的站点标题，可为空。

### 一言 / 随机句子
- `一言 (hitokoto)`：从 API 获取每日一句，默认调用 `https://v1.hitokoto.cn/?encode=js`。
- `自定义随机显示一句话`：手动维护句子列表，每次刷新随机展示。

两者可以同时启用，展示顺序：先显示自定义句子（若开启）再显示 hitokoto。

### 公告栏与多语言公告
- `个人简介/公告栏`：支持 HTML。
- `多语言个人简介/公告栏支持`：允许根据语言展示不同内容。

### 首页列表布局
- 可选 `简洁文章列表`、`多元文章列表`、`瞬间列表（需 plugin-moments）`。
- 每种布局都有独立开关（阅读量、分类、标签、摘要、封面等），简洁列表仅支持是否显示阅读量。
- `文章列表置顶图标` 与 `置顶图标位置` 控制置顶标识位置。

### 社交与标题展示
- `社交资料图标左侧文字`、`首页文章列表标题` 可单独隐藏，以满足简洁布局需求。

### 瞬间列表
- 仅在布局选择 `moment-list-summary` 时出现，可设定展示条数以及是否显示作者头像和昵称。

## 文章页样式

- `优化文章段落空行显示`：为段落添加最小高度，避免 Markdown 空行塌陷。
- `启用段落首行缩进`：可自定义缩进值（默认 2em）。
- `文章标题大写`：保留自上游主题的选项。
- `文章发布时间/更新时间`：可分别控制显示与左侧提示文字（发表于/更新于）。
- `显示文章阅读量`、`预计阅读时间`、`文章字数统计`：可结合 API 扩展插件获取更准确数据。
- `桌面端菜单中的分享按钮`：影响右侧浮动工具栏。
- `自定义侧边目录最大宽度`：默认 20rem，适合长标题的文章。
- `文章末尾的分隔线`：控制正文与下方模块之间的分隔。
- `文章底部的点赞按钮`：可配置尺寸、对齐、显示获赞数。
- `文章底部的推荐文章`：从当前文章第一个分类随机抽取，排除自身与不足数量的分类。
- `文章底部的相邻文章导航`：显示上一篇/下一篇链接。
- `文章评论区`：总开关，实际渲染仍依赖评论插件。
- `移动端底部导航栏` 与 `分享按钮`：控制移动端浮动菜单。

## 分类集合页样式

- `是否显示每个分类下的文章数量`：可自定义左右括号字符。
- `是否显示多层分类`：决定是否显示层级树。

## 分类详情页样式

- `文章列表显示文章阅读量`
- `显示 RSS 订阅按钮`：需安装 [plugin-feed](https://www.halo.run/store/apps/app-KhIVw)。

## 标签集合页样式

- `是否显示每个标签下的文章数量`：同样可自定义左右字符。
- `标签排序方式`：支持按文章数量、名称升降序等排序策略。

## 标签详情页样式

- `文章列表显示文章阅读量`
- `显示 RSS 订阅按钮`

## 作者详情页样式

- `显示 RSS 订阅按钮`：同样依赖 plugin-feed。

## 归档页样式

- `按照发布年份和月份折叠文章列表`：启用后可设置展开/折叠动画时长（毫秒）。

## 自定义页面样式

- 与文章页类似，可控制段落排版、预计阅读时间、字数统计等。
- `启用类文章页样式`：使自定义页面获得文章页的菜单、目录、回到顶部等功能。
- `是否启用正文内容与评论区间的分隔线`
- `是否启用评论区`

## 相册页样式（plugin-photos）

- 可配置图片圆角、渐入动画、瀑布流列数、间距及最小宽度。
- 当未启用瀑布流时，可单独显示分组标题。
- `进阶配置选项` 允许自定义 `onmouseover` 与 `onmouseout` 行为，适合开发者微调交互。

## 瞬间页样式（plugin-moments）

- 控制帖文预计阅读时间、字数统计、点赞按钮与评论区。点赞按钮默认开启。

## 社交资料 / RSS

### 首页社交资料展示
- 可添加预设平台、纯文本、自定义资料。对于 QQ、BiliBili 等平台，只需填写对应 ID；自定义资料则需在下方定义识别码、链接、图标与 aria-label。
- 若开启多语言菜单，可结合 `aria-label` 提供无障碍文本。

### 设定自定义资料
- 定义可复用的“识别码”。在“首页社交资料展示”中选择“自定义资料”后，填入该识别码即可。

## 自定义分享按钮

- `分享按钮设置` 列表可任意增删排序。
- URL 模板中支持 `@URL` 和 `@TITLE` 占位符，最终会替换为当前页面地址与标题。
- 预置了 Email、二维码、QQ、微信、微博、Telegram 等常用渠道。
- 如需禁用特定平台，直接移除对应条目即可。

## 错误页样式

- `页面自动重定向`：默认开启，等待 5 秒后跳转目标链接（默认 `/`）。可根据业务需求关闭或修改等待时间。
