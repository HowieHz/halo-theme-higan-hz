# 多语言实践

higan-hz 在原主题基础上强化了多语言体系，既提供语言自动跳转、菜单与公告的国际化，也支持在文章/分类/标签/页面级别为 HTML `lang` 属性设置独立值。本指南梳理所有相关表单与操作步骤，帮助你快速搭建真正的多语言站点。

> 说明文中的设置路径均使用 `分组 → 选项名称` 的形式表示。例如 `全局 → 默认页面语言`。

## 站点默认语言标识

- 路径：`全局 → 默认页面语言`
- 建议写入 ISO 或 BCP 47 语言代码（如 `zh`、`zh-CN`、`en-US`）。
- 留空时默认使用 `zh`。
- 访问者可通过在 URL 末尾追加 `?lang=代码` 的方式强制使用指定语言，例如 `https://example.com/?lang=en`。

## 多语言菜单

1. 在 `全局 → 多语言菜单支持` 勾选启用。
2. 建议同时启用 `全局 → 多语言功能前缀匹配模式`，允许以语言前缀匹配 `navigator.language` 值（如 `zh` 匹配 `zh_CN`、`zh_TW`）。
3. 进入 Halo 后台 **外观 → 菜单**，按以下结构构建：
   - 顶级菜单名称必须为语言代码，例如 `zh_CN` / `en_US`，或在启用前缀匹配后使用 `zh` / `en`。
   - 在对应语言的菜单下添加二级导航项（首页、关于等）。
4. 与 `全局 → 默认页面语言` 相同名称的菜单会作为默认回退菜单，在未匹配到语言时展示。

> 调试技巧：打开浏览器 DevTools，切换到 Console，执行 `navigator.language` 可以查看当前浏览器语言。

## 多语言公告栏（首页）

1. 在 `首页样式 → 多语言个人简介/公告栏支持` 勾选启用。
2. 在 `首页样式 → 自定义多语言公告栏内容` 列表中逐条添加：
   - **语言代码**：填写目标语言（如 `zh_CN` 或 `zh`）。
   - **内容**：支持 HTML，可嵌入链接、换行等。
3. 如果启用了 `多语言功能前缀匹配模式`，语言代码可以写成 `zh` / `en` 等前缀形式。
4. 与默认语言同名的条目会作为默认公告栏，在无法匹配语言时展示。

## 多语言页脚附加内容

1. 在 `总体样式 → 多语言页面最底部内容支持` 勾选启用。
2. 通过 `自定义多语言页面最底部内容` 列表分别配置各语言的 HTML 片段。
3. 支持与公告栏相同的前缀匹配逻辑。

## 浏览器语言自动跳转

- 路径：`全局 → 浏览器语言自动跳转`
- 启用后，访问者首次访问会按照浏览器语言自动跳转到匹配语言的页面。
- 可使用 `auto_redirect_target_language_list` 设置允许跳转的语言顺序（列表顶端优先）。
- 建议在站点已提供完整内容翻译时开启，避免跳转到未配置的路由。

## 元数据语言标识

为保证搜索引擎与无障碍工具识别正确语言，建议通过元数据为文章、分类、标签、自定义页面设置 HTML `lang`：

### 文章

- 后台路径：`内容 → 文章`，选择目标文章后点击右上角 **设置**，滚动至底部的元数据区域。
- 配置项：
  - **页面标题**：覆盖 HTML 标题（可选）。
  - **页面语言**：设置 HTML `lang` 属性，留空时回落至 `全局 → 默认页面语言`。
  - **显示在文章列表中**：可选，决定是否出现在列表聚合页。

### 分类

- 后台路径：`内容 → 文章 → 分类`，在列表中选择 **编辑**。
- 元数据配置与文章类似，可单独设定 **页面标题**、**页面语言**。

### 标签

- 后台路径：`内容 → 文章 → 标签`，选择 **编辑**。
- 同样可设置 HTML 标题与语言标识。

### 自定义页面

- 后台路径：`内容 → 页面`，进入目标页面后选择 **设置**。
- 支持页面标题与语言设置，与文章一致。

## 语言文件（固定文案）

动态表单之外，主题还允许修改内置文案：

1. 在 Halo 服务器的主题目录中找到 `themes/howiehz-higan/i18n`。
2. 针对不同语言编辑 `.properties` 文件（例如简体中文为 `zh.properties`）。
3. 保存后重新部署/刷新站点即可生效。

> 建议在修改语言文件前备份原文件，以便后续升级主题时进行合并。

## 调试与排错

- **菜单/公告未生效**：确认语言代码拼写是否与浏览器语言匹配，或启用前缀匹配模式。
- **自动跳转异常**：确保跳转目标在 `auto_redirect_target_language_list` 中，同时存在对应菜单与内容。
- **页面语言不正确**：检查是否已在元数据设置 `页面语言`；如未设置会回退到默认语言。
- **语言文件无效**：确保修改的是当前启用主题目录下的 `.properties` 文件，并已清理缓存。

完成上述配置后，就能为访问者提供全站一致的多语言体验。更多多语言相关示例可参考《[内容写作与增强](./content-authoring.md)》中的“多语言菜单与文案”章节。
